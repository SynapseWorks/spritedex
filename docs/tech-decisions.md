# Technical Decisions

## Database
PostgreSQL + PostGIS chosen for:
- GIS support
- scalability
- spatial querying

## Backend
FastAPI chosen for:
- speed
- async support
- Python ecosystem

## Frontend
Next.js chosen for:
- React ecosystem
- scalability
- component architecture

## Maps
MapLibre/Leaflet chosen for:
- open source flexibility
- GIS compatibility

## AI Strategy
API-first approach initially.
Local AI planned later.
