# SpriteDex Backend

The backend powers:
- species management
- encounter logging
- AI identification processing
- GPS and map queries
- media management
- API access
- authentication
- future AI integrations

---

# Planned Stack

- Python
- FastAPI
- PostgreSQL
- PostGIS
- SQLAlchemy
- Alembic
- Redis (future)
- Celery/RQ (future)

---

# Planned Responsibilities

## Species System
Manage canonical species data.

## Encounter System
Store and query personal observations.

## Identification System
Handle AI species candidate suggestions.

## Spatial System
Support GPS and geographic queries.

## Media System
Store encounter media metadata.

## Future AI Layer
Support local and cloud AI integrations.
